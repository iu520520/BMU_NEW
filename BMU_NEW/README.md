# BMU_NEW Firmware

Recommended source layout:

- `BMU_NEW/`: code generated by NTFx, including the Keil project and vendor drivers
- `app/`: application entry points called from generated `main.c`
- `bsp/`: board-specific pin and hardware mapping
- `drivers/`: hand-written drivers, such as software I2C and MP2797
- `services/`: measurement snapshots and validation
- `safety/`: protection logic and output arbitration
- `platform/`: toolchain/runtime support

Keep hand-written BMS logic outside `BMU_NEW/` as much as possible. NTFx regeneration may overwrite files under `BMU_NEW/USER/`.
